What is Scatter

Scatter is a way to sign transactions and provide private data without exposing keys and unnecessary information while interacting with web applications communicating with the EOS Blockchain.

Take a shopping website as an example. Normally you have to click buy, then enter your private keys / credit card, shipping address, email, and any other information they require in order to ship your purchase to you, then you have to create an account to track your order which again requires your email and a password. With Scatter you set up an Identity once, and then simply click buy. Applications can then request shipping information right from the transaction signature prompt.

Identities inside of Scatter are the containers for personal information and private data. You configure them once and use them as much as you want. They provide a layer of abstraction away from your EOS accounts and help get rid of the need for pesky multi-page registrations.

Inside of them you store anything that an application might need to use in the future, for instance:

  • Identity Name ( the username for applications )

  • EOS Accounts / Key Pairs

  • Personal Information ( full name, email, date of birth )

  • Location Information ( phone, address, city, state, country, zip code )

A single Scatter can hold multiple Identities which allows for multiple sets of data points. For instance if you want to purchase a gift for your spouse you can set up an Identity for them inside of your Scatter once and use it over and over again without having to re-enter that information.

Absolutely nothing can be retrieved from your Scatter without some sort of Permission.
There are two types of permissions within Scatter.

Identity Permissions
These are the initial authentication permissions you provide to an application. Without an Identity Permission an application can not retrieve any information or request any transaction signatures.

Contract Action Permissions
These are optional permissions based on signature requests. They allow you to check a box from the signature request prompt to sign the same transaction automatically the next time it is requested. This is great for things like chat style applications for instance as it would be terrible user experience to have to click not only the submit button on the application every time, but also approve the transaction.




If you're going to sign something, you have the right to know exactly what you're signing. With Scatter we make sure you always do. You can view the transaction you are about to sign as either a list of human-readable properties or as the raw JSON that gets sent to the blockchain.

Scatter also allows applications to request other information from you when signing transactions as well; such as shipping information. If you look at the image above you can see that the application requires not only an EOS account but also firstname, lastname, country, address, and city. When you accept the transaction signature request those fields from the Identity ( located top right: Clark Kent ) will also be returned.

Since a lot of those properties are from location information there's also a dropdown there which will allow you to select which location ( since an Identity can have many, like home or work. ) you want to provide shipping information from.

At the very bottom you can also see a section for whitelisting this transaction by adding a Contract Action Permission. If you do this you will not have to re-accept this signature request next time as long as every property matches.
